LLM Gateway — 개발자 설정 가이드

사내 LLM 게이트웨이를 통해 Claude Code 와 Codex 에서 승인된 모델을 사용하는 방법입니다. 관리자가 배포한 값으로 채워져 있습니다.

항목값
게이트웨이 URLhttps://llmgw.example.com
리전us-east-1
사용 가능 모델claude-opus · claude-sonnet · claude-haiku · claude-fable · gpt-5.5
로그인 방식Cognito Hosted UI (llmgw-login) authMode에 따라 상이
가상 키(virtual key)는 로그인 후 토큰 헬퍼가 자동 발급/회전합니다. 키를 직접 붙여넣지 마세요 — 아래 설정이 헬퍼를 통해 자동으로 주입합니다.

1 로그인 (가상 키 발급 준비)

authMode: cognito-native (Org SSO 미사용 환경)

llmgw-login 실행 → 브라우저에 Cognito Hosted UI(이메일/비밀번호)가 열립니다. 로그인하면 토큰이 로컬에 캐시됩니다. AWS CLI·자격증명 불필요.

llmgw-login          # Windows: llmgw-login.ps1
authMode: org-sso (IdC 조직 인스턴스)

SSO 프로필로 로그인합니다. 토큰 헬퍼가 SigV4로 서명합니다.

aws sso login --profile llmgw

2 게이트웨이 접속 certMode: http 일 때만

도메인 없는 PoC 배포(certMode: http)는 게이트웨이가 HTTP(평문) 로 서비스됩니다 — 가상 키와 프롬프트가 암호화 없이 전송되므로 PoC 용도로만 사용하세요. 접근은 허용된 사내 IP 대역(Security Group allowlist)에서만 가능합니다. 별도의 인증서 신뢰나 터널 설정은 필요 없습니다.

certMode: acm(공인 인증서) 환경이면 이 단계는 건너뜁니다 — 공인 신뢰라 아무 설정도 필요 없습니다. http 환경에서 접속이 안 되면 현재 네트워크의 공인 IP가 관리자에게 등록된 허용 CIDR에 포함되는지 확인하세요.

3 Claude Code 설정

~/.claude/settings.json 에 아래를 병합합니다. 4개 모델 별칭을 모두 넣어야 /model 선택기에 Fable 티어까지 노출됩니다.

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://llmgw.example.com",
    "AWS_REGION": "us-east-1",
    "ANTHROPIC_DEFAULT_OPUS_MODEL":   "claude-opus",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "claude-sonnet",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL":  "claude-haiku",
    "ANTHROPIC_DEFAULT_FABLE_MODEL":  "claude-fable"
  },
  "apiKeyHelper": "~/.llmgw/gateway_auth.sh apiKey",
  "permissions": { "deny": ["WebSearch"] }
}
permissions.deny: ["WebSearch"] 로 Claude 내장 WebSearch를 끕니다 — 대신 아래 5단계의 관리형 웹 검색 MCP를 사용합니다.

4 Codex 설정

~/.codex/config.toml 에 아래를 추가합니다.

model = "gpt-5.5"

[model_providers.llmgw]
name     = "llmgw"
base_url = "https://llmgw.example.com/v1"
wire_api = "responses"
web_search = "disabled"

[model_providers.llmgw.auth]
command = "~/.llmgw/gateway_auth.sh token"

5 웹 검색 MCP 등록 (Claude Code)

서버 측 등록만으로는 클라이언트에서 안 보입니다. headersHelper 로 회전 키를 동적 주입하며 등록하세요.

claude mcp add-json websearch '{
  "type": "http",
  "url": "https://llmgw.example.com/mcp/",
  "headersHelper": "~/.llmgw/gateway_auth.sh mcp-headers"
}'

등록 후 도구 이름: websearch-web-search-tool___WebSearch

6 동작 확인

curl -s https://llmgw.example.com/v1/models \
  -H "Authorization: Bearer $(~/.llmgw/gateway_auth.sh token)"
# 기대: claude-opus/sonnet/haiku/fable + gpt-5.5 목록

이제 claude / codex 를 실행하면 호출이 게이트웨이로 나가고, 사용량은 관리자 대시보드에 집계됩니다.

문제가 있으면 게이트웨이 관리자에게 문의하세요. 이 문서에는 관리자 비밀·마스터 키가 포함되지 않습니다 — 여러분의 가상 키는 로그인으로 자동 발급됩니다.
(예시 값으로 렌더링된 미리보기입니다. 실제 배포 시 관리자가 배포값으로 채운 문서를 받게 됩니다.)
