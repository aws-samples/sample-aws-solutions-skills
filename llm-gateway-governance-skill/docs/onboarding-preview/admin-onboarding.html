3 비밀번호 / 키 변경 방법

LiteLLM 마스터 키 회전

마스터 키는 Secrets Manager에 있고 컨테이너 시작 시 로드됩니다. 값 갱신 후 ECS 강제 재배포로 반영합니다.

aws secretsmanager put-secret-value \
  --secret-id llmgw-litellm-admin-key --secret-string 'sk-새로운마스터키'
aws ecs update-service --cluster llmgw-litellm --service litellm --force-new-deployment
개발자 가상 키는 별도 DB 레코드라 마스터 키 회전에 영향받지 않습니다. Admin UI 재로그인만 새 키로 하면 됩니다.

Langfuse 관리자 비밀번호

Langfuse UI 로그인 → Settings → Account 에서 변경이 가장 간단합니다. (초기값은 위 Secrets Manager 시드)

Cognito 사용자 비밀번호

# 관리자 강제 재설정
aws cognito-idp admin-set-user-password \
  --user-pool-id us-east-1_ABC123XYZ --username dev@example.com --password '새비밀번호' --permanent
# 사용자 셀프 재설정: Hosted UI의 "Forgot password"

4 개발자 온보딩 (관리자 작업)

cognito-native — 사용자 생성 + 팀 그룹 배정

aws cognito-idp admin-create-user --user-pool-id us-east-1_ABC123XYZ \
  --username dev@example.com --user-attributes Name=email,Value=dev@example.com \
  --desired-delivery-mediums EMAIL
aws cognito-idp admin-add-user-to-group --user-pool-id us-east-1_ABC123XYZ \
  --username dev@example.com --group-name llmgw-team1   # 그룹명 = LiteLLM 팀

org-sso — permission set 할당

IdC에서 해당 permission set을 사용자/그룹에 할당합니다. 신규 팀 = 신규 permission set + LiteLLM Admin UI 팀 설정(코드 재배포 불필요).

그다음 개발자에게 developer-setup.html 을 전달하면 끝입니다.

5 오프보딩 (접근 회수)

cognito-native

# 비활성(복구 가능) / 삭제 / 팀에서만 제거 중 택1
aws cognito-idp admin-disable-user --user-pool-id us-east-1_ABC123XYZ --username dev@example.com
aws cognito-idp admin-delete-user  --user-pool-id us-east-1_ABC123XYZ --username dev@example.com
aws cognito-idp admin-remove-user-from-group --user-pool-id us-east-1_ABC123XYZ --username dev@example.com --group-name llmgw-team1

org-sso

IdC에서 permission set 할당을 해제합니다.

⏱ 즉시 차단하려면 LiteLLM Admin UI에서 해당 가상 키를 revoke 하세요. 키는 DynamoDB 캐시에 TTL 동안 남아, revoke 없이는 로그인 차단 후에도 기존 키가 TTL 만료까지 동작할 수 있습니다.
